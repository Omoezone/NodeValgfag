Variables
Variables in NodeJS has the same functions as in most other languages. It is an container that hold values.
We use 3 different kind of variables. Var, let and const. We will distinguish between mutable and imutalbe variables.
A mutable variable is one, that can be modified after it is created, where an imutable can not. 

> var & let: Both are mutable and is much alike with some differences. The main difference is the scope of the variables. 
Var has it scope to functions, while let has block scope ie. to enclosing brackets ("{}"). Example: 

            function run() {
              var foo = "Foo";
              let bar = "Bar";
            
              console.log(foo, bar); // Foo Bar
              {
                var moo = "Mooo"
                let baz = "Bazz";
                console.log(moo, baz); // Mooo Bazz
              }
              console.log(moo); // Mooo
              console.log(baz); // ReferenceError
            }
            run();

Here is the var scoped to the entire function, where let is only usable inside the enclosing brackets. 
So the var in the middle ("Mooo") is used outside of the enclosing brackets as it log it in the end, with no problems
as it is still inside of the function run(), but it throws an error for the let, as the console log
for the let ("Bazz") is outside of the scope of the enclosing brackets ("{}").

> Const: Is an imutable variable. Const works a lot like let which we have just talked about, with the 
exception that it cant be changed after it has been created. That it works like let means that it also has block scope.
Same example as above can be used with let being swapped with const.

          
Scope

          
Scope is the extend of which a variable is visible to the program. All variable have a scope but not all variables
have the same type of scope. we primarily work with 2 different types of scope. Function- and blocks scope.
> function scope:  Where the variable is visible inside of an entire function.
> block scope: Where the variable is visible inside of the enclosing brackets it is inside. 


Hoisting

Hoisting is an important thing to be aware of when you talk about scope. Hoisting is the name of an feature in javascript That
that declares every variable inside a given scope in the beginning of the function. It does this even if the variable first 
is declared later down the line of the function. Example: 

          function f() {
            console.log(bar);  // undefined
            var bar = 'abc';
            console.log(bar);  // abc
          }
So the bar variable is declared already before the first console.log expression but returns undefined as there is no value to bar yet.
