Express

On this page will we be looking at express and how we ready an express projects.

Before we cant begin the work with our NodeJS programs, we might want to install and use express. 
      

          
1. Create a project folder

          
You simply need to make a folder, which we can then use in a workspace. When we are working with express, would we want to 
make a Javascript file called app.js (standard) and a JSON file called package.json to create dependencies.

          
2. Create a package.json file (create dependencies)

          
Create a new file and name it (name).json. Then we new to ensure that we actually make it a Json by using the "{}". 
We are now ready to declare the dependencies for the project. When we do that, should we remember to think "key & values". 
First we need the key which is "dependencies": and the value is what is inside the "{}". Inside of those, will we have a new key & value pair. 
As we are wanting to work with express, will the next key be "express": and the value will be the version of the key (express) we will be running. 
So the final result would be:
{
  "dependencies": {
      "express": "^4.17.1"
  }
}
We can then add more dependencies inside of the tags in the key & value format.

          
3. Install express for our project with npm install

          
We have now made the dependencies for our NodeJS project but we have not yet installed express for our project, which we will do now.
For that would we need to use npm(Node Package Manager) to install express. We do that by a terminal command called npm install or just npm i.
This command looks in our created package.json file and install the dependencies that we have defined, which in this case is only express. 
This command should be the first thing you do, when you recieve a NodeJS program from another person, to ensure that you run the program with the correct dependencies.


          
4. Tell NodeJS that we want to use express (require)

          
As we have now installed express and other eventual dependencies, will we now have to "tell" our app.js (Node program) that we want to use express.
We do this by first importing the express module (that we just installed) from our newly generated node_modules folder, with the require funtion.
How we do this, is showned here: 
      const express = require('express');
      const app = express();

We create and constant called express which we then use to hold the express module.
Then we create a new constant called app, which would be the constant we use to create our callbacks.    

          
5. Create a listen callback

          
The last step we need to do before we can use our program. We need to define how the program communicates. 
We do this with the listen function which we call on our app constant, we just created. 
The listen function will be a callback, that takes the port we run the program on and optionally an error. It is this function that enable us to run the program. 
A good practise would be to tell the user that the program is running and at what port in the console. Example: 

      app.listen(8080, (error) => {
          if(error){
              console.log("The program experienced an error" + error);
          }else {
              console.log("Starting program on port: ", 8080);
          }
      });
