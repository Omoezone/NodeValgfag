Terminal

We will on this page demonstrate how we work with the terminal
as we use the different tools in our toolbelts, such as git commands and npm.
      

          
Git commands

          
When we are coding and working with IDE's, will we often use github as a file manager and sharing tool. 
  To do this will we need understand the different commands we have to use. we do this in a few steps.
  All steps need to begin with keyword "git".
  1. We need to save the files that we need to be sent to github.                  
      > git add -A (The "A" says 'save everything in the project'. it can be more specifik too)
  2. We need to pack the files in a commit that is ready to be sent to our github. 
      > git commit -m "Here we put commit message"
  3. We then need to send the files to our github and finalize the commit, with a push. 
      > git push

          
Node & Nodemon

          
Node and nodemon is terminal commands that we use to run our program in its current state. 
  It is these commands that trigger the listen function in our program. 
  The node command runs the program once, while nodemon keeps restarting the program each time we change something i our project.
  Node is preinstalled and a part of the basic nodeJS framework, where we need to install nodemon ourself with a npm command. 
  
  Usage: 
  We call either node or nodemon on the file we want to run. We made an file called app.js earlier which we
  now want to run with the help of the terminal. We ensure that we are on the correct path (the folder where we saved app.js).
  Then we simply write either node or nodemon followed by the name of the file we want to run, like this:
      > PS C:\Users\xxx\xxx\NodeJS\NodeValgfag\Mandatory\Mandatory1> nodemon.cmd app.js
      or
      > PS C:\Users\xxx\xxx\NodeJS\NodeValgfag\Mandatory\Mandatory1> node app.js
  We will sometimes have to put .cmd after node or nodemon to be able to use it. That is a path problem which is easily fixed this way. 

          
npm

          
npm (Node Package Manager) is the default package manager for javascript. It is an online database of packages, that we
  (the users) can access and use. This online database is called the npm registry. We access the npm registry with our client in the terminal
  and install the given packages we need and/or want. We can find all sorts of different packages on the npm website.
